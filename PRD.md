# 할 일 관리 웹페이지 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로젝트명
개인용 할 일 관리 웹페이지 (To-Do List)

### 1.2 목적
매일 10~20개의 할 일을 손쉽게 추가·관리하고, 카테고리와 진행 상태별로 분류하여
하루의 일과를 효율적으로 관리할 수 있는 개인용 웹페이지를 제공한다.

### 1.3 대상 사용자
- 개인 일정과 업무를 관리하려는 단일 사용자 (본인 전용)
- 별도의 로그인 없이 즉시 사용하기를 원하는 사용자

### 1.4 기술 스택 및 제약
- **사용 기술**: HTML, CSS, 순수 JavaScript (Vanilla JS)
- **실행 환경**: 웹 브라우저에서 파일 열기만으로 바로 실행
- **제외 범위**: 회원가입, 로그인, 서버/백엔드, 데이터베이스
- **구현 방향**: 외부 라이브러리·프레임워크 없이 기술적으로 단순하게 구현

---

## 2. 주요 기능

### 2.1 할 일 관리 (CRUD)
| 기능 | 설명 |
|------|------|
| 추가 | 입력창에 내용과 카테고리를 지정해 새 할 일 등록 |
| 수정 | 기존 할 일의 내용 및 카테고리 변경 |
| 삭제 | 개별 할 일 제거 |

### 2.2 완료 체크
- 각 할 일에 체크박스 제공
- 체크 시 완료 상태로 전환 (취소선 등 시각적 표시)
- 다시 클릭하면 진행 중 상태로 복귀

### 2.3 카테고리 분류
- 3가지 카테고리: **업무 / 개인 / 공부**
- 할 일 추가·수정 시 카테고리 선택
- 카테고리별 색상 또는 라벨로 구분 표시

### 2.4 상태 필터
- **전체 / 진행 중 / 완료** 3가지 필터 제공
- 선택한 필터에 해당하는 할 일만 목록에 표시

### 2.5 완료율 표시
- 전체 할 일 대비 완료된 할 일의 비율(%) 계산 및 표시
- 진행 상황을 시각적으로 확인할 수 있도록 진행 바 또는 숫자로 표현
- 할 일 추가/삭제/완료 시 실시간 갱신

## 3. 화면 구성

```
┌─────────────────────────────────────────────┐
│              할 일 관리 (제목)                │
├─────────────────────────────────────────────┤
│  완료율: [■■■■■□□□□□] 50%  (5/10)            │  ← 진행률 영역
├─────────────────────────────────────────────┤
│  [할 일 입력......] [카테고리▼] [추가 버튼]   │  ← 입력 영역
├─────────────────────────────────────────────┤
│  [전체] [진행 중] [완료]                      │  ← 필터 영역
├─────────────────────────────────────────────┤
│  ☐ 보고서 작성        [업무]  [수정][삭제]    │
│  ☑ 운동하기          [개인]  [수정][삭제]    │  ← 할 일 목록
│  ☐ 영어 단어 암기     [공부]  [수정][삭제]    │
│  ...                                          │
└─────────────────────────────────────────────┘
```

### 3.1 구성 요소
1. **헤더 영역**: 페이지 제목
2. **진행률 영역**: 완료율(%) 및 진행 바, "완료/전체" 개수 표시
3. **입력 영역**: 할 일 텍스트 입력창, 카테고리 선택, 추가 버튼
4. **필터 영역**: 전체 / 진행 중 / 완료 버튼
5. **목록 영역**: 할 일 항목 리스트 (체크박스, 내용, 카테고리 라벨, 수정·삭제 버튼)

## 4. 데이터 저장 방식

### 4.1 저장소
- 브라우저의 **localStorage** 사용
- 서버 없이 사용자의 브라우저에 데이터를 영구 저장 (새로고침·재방문 시 유지)

### 4.2 데이터 구조 (예시)
```javascript
// localStorage 키: "todos"
[
  {
    "id": 1719100000000,   // 고유 식별자 (생성 시각 기반)
    "text": "보고서 작성",  // 할 일 내용
    "category": "업무",     // 업무 / 개인 / 공부
    "completed": false      // 완료 여부
  },
  ...
]
```

### 4.3 저장 시점
- 할 일 추가 / 수정 / 삭제 / 완료 상태 변경이 발생할 때마다 localStorage에 저장
- 페이지 로드 시 localStorage에서 데이터를 읽어와 화면에 렌더링

## 5. 완료 기준 (Definition of Done)

다음 항목이 모두 충족되면 프로젝트를 완료로 간주한다.

- [ ] HTML, CSS, 순수 JavaScript만으로 구현되었다.
- [ ] 별도 설치·서버 없이 브라우저에서 파일을 열어 바로 실행된다.
- [ ] 할 일을 추가, 수정, 삭제할 수 있다.
- [ ] 할 일의 완료 여부를 체크/해제할 수 있고 시각적으로 구분된다.
- [ ] 업무 / 개인 / 공부 카테고리로 분류할 수 있다.
- [ ] 전체 / 진행 중 / 완료 필터가 정상 동작한다.
- [ ] 전체 대비 완료율(%)이 표시되고 실시간으로 갱신된다.
- [ ] localStorage에 데이터가 저장되어 새로고침·재방문 후에도 데이터가 유지된다.
- [ ] 10~20개의 할 일을 무리 없이 관리할 수 있다.
- [ ] 기본적인 예외 처리(빈 입력 방지 등)가 되어 있다.
