# CLAUDE.md

이 파일은 새 Claude Code 세션이 이 프로젝트를 빠르게 이해하기 위한 안내서입니다.

## 프로젝트 개요

개인용 **할 일 관리 웹페이지(Tasky)**. 하루 10~20개의 할 일을 추가·관리하고
카테고리·마감일·진행 상태별로 정리하는 단일 사용자용 앱.

- **기술 스택**: HTML + CSS + 순수 JavaScript (Vanilla JS)
- **외부 의존성 없음**: 프레임워크·라이브러리·빌드 도구·서버 모두 사용하지 않음
- **실행**: `index.html`을 브라우저에서 열기만 하면 동작 (설치/서버 불필요)
- **저장**: 브라우저 `localStorage` (키: `"todos"`) — 서버·DB 없음

관련 문서: [PRD.md](PRD.md)(요구사항), [PLAN.md](PLAN.md)(구현 흐름)

## 파일 구조

```
001-study/
├─ index.html          # 화면 구조 (사이드바 + 요약 카드 + 툴바 + 목록)
├─ style.css           # 전체 스타일 (대시보드 톤, 반응형)
├─ script.js           # 모든 로직 (CRUD, 필터, 정렬, 통계, 저장)
├─ PRD.md              # 제품 요구사항 정의서
├─ PLAN.md             # 구현 단계 계획
├─ CLAUDE.md           # (이 파일)
└─ backup/
   └─ original/        # 디자인 개편 전 원본 백업 — 수정/삭제 금지
```

> **주의**: `backup/original/` 안의 파일은 초기 버전 백업이므로 **수정하거나 삭제하지 않는다**.

## 기능 목록

- 할 일 추가 / 수정 / 삭제 / 완료 체크
- 카테고리 분류: **업무 / 개인 / 공부**
- 상태 필터(전체 / 진행 중 / 완료) + 카테고리 필터(사이드바)
- 마감일 입력 및 색상 표시: 기한 초과(빨강) / 오늘(주황) / 내일(노랑) / 여유(회색)
- 상단 요약 카드: 전체 / 진행 중 / 완료 / 완료율(진행 바)
- 실시간 검색, 정렬(등록순 / 마감일 임박순 / 카테고리순)
- 완료 항목 일괄 삭제
- 모바일 반응형(사이드바 햄버거 토글)

## 구현 방식 (script.js 핵심)

- **상태 변수**: `todos`(배열), `statusFilter`, `categoryFilter`, `searchTerm`, `sortBy`
- **데이터 모델**: 할 일 1건 =
  ```js
  { id, text, category, completed, due, createdAt }
  ```
  - `id`, `createdAt`: `Date.now()` 기반 / `due`: `"YYYY-MM-DD"` 문자열 또는 `""`
- **렌더링 흐름**: 모든 변경은 `상태 수정 → saveTodos() → render()` 순서.
  `render()`는 `getVisibleTodos()`(필터+검색+정렬)로 목록을 만들고 DOM을 다시 그린 뒤 `updateStats()` 호출.
- **저장/복원**: `saveTodos()`/`loadTodos()`가 `localStorage`와 동기화. 로드 실패 시 빈 배열로 폴백.
- **마감일 색상**: `describeDue(due)`가 `dayDiff()`로 오늘과의 일수 차를 계산해 라벨+클래스 반환.
- **DOM 생성**: `createTodoElement(todo)`가 항목 하나의 `<li>`를 만들고 이벤트를 직접 바인딩(템플릿 문자열 innerHTML 아님 → XSS 안전).

## 수정 시 주의할 점

- **순수 JS 유지**: 프레임워크·빌드 도구·외부 라이브러리를 도입하지 않는다. (PRD 제약)
- **데이터 모델 변경 시 하위 호환**: `loadTodos()`로 불러온 기존 데이터에는 새 필드가 없을 수 있다.
  필드를 추가하면 읽는 쪽에서 기본값(`todo.due || ""` 등)을 처리할 것.
- **`localStorage` 키 `"todos"` 고정**: 키를 바꾸면 기존 사용자 데이터가 사라진다.
- **변경 후 항상 `saveTodos()` + `render()`**: 둘 중 하나라도 빠지면 화면 또는 저장이 어긋난다.
- **사용자 입력은 `textContent`로 출력**: `innerHTML`에 직접 넣지 말 것(보안).
- **빈 입력 방지** 등 기존 예외 처리를 유지한다.
- **카테고리 값은 한글 문자열**(`업무/개인/공부`)이며 CSS 클래스(`tag--업무` 등)와 직접 연결되어 있다.
  카테고리를 바꾸면 HTML·CSS·JS 세 곳을 함께 수정해야 한다.

## 실행 / 점검 방법

1. `index.html`을 브라우저에서 더블클릭으로 연다.
2. 할 일을 서로 다른 카테고리·마감일로 몇 개 추가하고 일부를 완료 체크한다.
3. 검색·정렬·필터·완료율 동작을 확인한다.
4. **F5 새로고침** 후 항목과 완료 상태가 그대로 유지되는지(localStorage) 확인한다.

> 이 환경에는 Node.js가 없어 자동 테스트 도구는 없다. 검증은 브라우저에서 수동으로 한다.
