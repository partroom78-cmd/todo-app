# 할 일 관리 웹페이지 구현 계획 (PLAN)

> 참고 문서: [PRD.md](PRD.md)
> 기술 스택: HTML + CSS + 순수 JavaScript (외부 라이브러리·서버 없음)

---

## 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 화면 구조(마크업) 및 CSS·JS 파일 연결 |
| `style.css`  | 레이아웃, 색상, 카테고리 라벨, 완료 표시 등 스타일 |
| `app.js`     | 데이터 처리, 렌더링, 이벤트 처리 등 모든 기능 로직 |

> 단순함을 위해 HTML/CSS/JS를 3개 파일로 분리한다.
> (원한다면 하나의 `index.html`로 합칠 수도 있으나, 유지보수를 위해 분리 권장)

---

## 1단계 — 화면 구성 (정적 마크업)

**목표:** 동작 없이 화면의 뼈대를 먼저 만든다. (PRD 3. 화면 구성 기준)

- [ ] 헤더 영역: 페이지 제목
- [ ] 진행률 영역: 완료율(%) 텍스트 + 진행 바 + "완료/전체" 개수 자리
- [ ] 입력 영역: 할 일 입력창, 카테고리 선택(select), 추가 버튼
- [ ] 필터 영역: 전체 / 진행 중 / 완료 버튼
- [ ] 목록 영역: 할 일 항목이 들어갈 빈 컨테이너(`<ul>`)
- [ ] 기본 스타일 적용 (레이아웃, 카테고리 색상, 완료 시 취소선 등)

**만들 파일:** `index.html`, `style.css`

---

## 2단계 — 데이터 저장 기반 만들기

**목표:** 기능 구현 전에 데이터 구조와 저장/불러오기 토대를 먼저 잡는다. (PRD 4. 데이터 저장 방식 기준)

- [ ] 할 일 데이터를 담을 배열(`todos`) 정의
- [ ] 데이터 구조 확정: `{ id, text, category, completed }`
- [ ] `localStorage` 저장 함수 (`saveTodos`)
- [ ] `localStorage` 불러오기 함수 (`loadTodos`)
- [ ] 페이지 로드 시 저장된 데이터를 읽어와 초기화

**만들 파일:** `app.js`

---

## 3단계 — 기능 구현 (핵심 로직)

**목표:** PRD 2. 주요 기능을 하나씩 동작하게 만든다. 각 동작 후 저장 + 화면 갱신.

### 3-1. 렌더링 함수
- [ ] `todos` 배열을 받아 목록 영역에 그리는 `renderTodos` 함수
- [ ] 데이터 변경 시마다 다시 그려 화면을 최신 상태로 유지

### 3-2. 할 일 추가 (Create)
- [ ] 입력값 + 선택 카테고리로 새 항목 생성 후 배열에 추가
- [ ] 빈 입력 방지 등 예외 처리

### 3-3. 완료 체크 (Toggle)
- [ ] 체크박스 클릭 시 `completed` 상태 전환
- [ ] 완료 항목 시각적 표시(취소선 등)

### 3-4. 수정 (Update)
- [ ] 기존 항목의 내용·카테고리 변경

### 3-5. 삭제 (Delete)
- [ ] 개별 항목 제거

### 3-6. 상태 필터
- [ ] 전체 / 진행 중 / 완료 필터에 따라 보여줄 항목 선별
- [ ] 현재 선택된 필터 표시

### 3-7. 완료율 표시
- [ ] 전체 대비 완료 개수로 완료율(%) 계산
- [ ] 진행 바 + 숫자(완료/전체) 갱신
- [ ] 추가/삭제/완료 변경 시 실시간 반영

**수정 파일:** `app.js` (필요 시 `index.html`, `style.css` 보완)

---

## 4단계 — 최종 점검

**목표:** PRD 5. 완료 기준(Definition of Done) 항목을 하나씩 검증한다.

- [ ] 추가 / 수정 / 삭제 정상 동작 확인
- [ ] 완료 체크/해제 및 시각적 구분 확인
- [ ] 카테고리(업무/개인/공부) 분류 표시 확인
- [ ] 전체 / 진행 중 / 완료 필터 정상 동작 확인
- [ ] 완료율(%) 실시간 갱신 확인
- [ ] 새로고침·재방문 후 데이터 유지(localStorage) 확인
- [ ] 빈 입력 등 예외 처리 확인
- [ ] 10~20개 항목에서 무리 없이 동작하는지 확인
- [ ] 브라우저에서 파일 열기만으로 바로 실행되는지 확인

**점검 대상 파일:** `index.html`, `style.css`, `app.js`

---

## 구현 순서 요약

```
1. 화면 구성   →  index.html, style.css   (정적 화면 뼈대)
2. 데이터 저장 →  app.js                  (localStorage 토대)
3. 기능 구현   →  app.js                  (CRUD·완료·필터·완료율)
4. 최종 점검   →  전체 파일               (완료 기준 검증)
```
